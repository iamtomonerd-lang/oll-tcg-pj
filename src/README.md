# TCG汎用ゲームエンジン - アーキテクチャ

複数のTCGタイトルを実装できる汎用TCGゲームエンジン。

---

## 全体構成

```
        汎用パーツ
        ↙ ↓ ↓ ↘
     層0 層1 層2 ゲーム側
```

---

## 各要素の責務

### 層0：概念・型定義

**役割：** 「何が存在するか」を定義する

**内容：**
- カード型
- プレイヤー型
- ゾーン型
- ゲームオブジェクト型

**特性：**
- ゲーム非依存（TCGタイトルに依存しない）
- 実行処理を含まない
- すべてのTCGで共通

**責務でないこと：**
- ❌ カードの移動方法
- ❌ プレイヤー固有の属性
- ❌ ゲーム固有の型

---

### 層1：実行処理

**役割：** 「実際にどう処理するか」を実装する

**内容：**
- 共通処理（検索、ソート、フィルタなど）
- カード処理（属性変更、状態管理など）
- プレイヤー処理（情報管理など）
- ゾーン処理（追加、削除、検索など）
- ゲームオブジェクト処理

**特性：**
- ゲーム非依存（汎用的な処理のみ）
- 型安全（層0の型を利用）
- 副作用なし（イミュータブル）

**責務でないこと：**
- ❌ 処理の順番管理
- ❌ ゲームルール判定
- ❌ 効果解決

---

### 層2：処理管理・監視

**役割：** 「処理の順番を管理し、状態変化を通知する」

**内容：**

1. **処理エンジン**
   - 処理を順番に実行
   - FIFO キュー管理
   - 同期・非同期処理対応

2. **効果解決**
   - 効果を処理に分解
   - 処理に変換

3. **状態変化管理**
   - 状態変化を記録
   - リスナーに通知
   - イベント生成

**特性：**
- ゲーム非依存（純粋なエンジン）
- 処理順序を保証
- 状態変化の追跡可能
- イミュータビリティ保証

**責務でないこと：**
- ❌ ゲームルール判定
- ❌ 効果内容の決定
- ❌ カード移動ルール

---

### 汎用パーツ：共有資産

**役割：** 「複数システムで再利用可能な部品を提供する」

**位置づけ：** 階層ではなく、すべての層が利用できる資産

**内容：**
- 数値管理
- 条件管理
- データ管理
- ゾーン管理
- オブジェクト管理
- リソース管理

**特性：**
- ゲーム非依存
- 複数利用場所を想定
- 再利用可能

**利用者：**
- 層0, 層1, 層2
- ゲーム側
- UI側
- サーバー側

**責務でないこと：**
- ❌ ゲーム固有ルール
- ❌ 特定ゲーム用語
- ❌ ゲーム固有判定

---

### ゲーム側（層3+）：意味付け・ルール定義

**役割：** 「パーツに名前・ルール・用途を付与する」

**内容：**
- パーツの具体化（「数値管理」→「ライフ管理」）
- ゲームルールの実装
- 条件判定の具体化
- 勝敗判定
- ターンシステム
- カード効果定義

**特性：**
- ゲーム固有（特定のTCGタイトル向け）
- 層2以下のエンジンに依存
- 複数存在（ゲームタイトル数分）

**利用：**
- 層0, 層1, 層2 のすべてを利用
- 汎用パーツを利用
- 独自ルールを定義

---

## 責務の分離

| 判断軸 | 回答 | 配置 |
|-------|------|------|
| 処理を管理するか？ | はい | 層2 |
| 実際に処理するか？ | はい | 層1 |
| ゲームごとに意味が変わるか？ | はい | ゲーム側 |
| 複数で再利用できるか？ | はい | 汎用パーツ |

---

## 設計原則

### 1. ゲーム非依存性

層0・層1・層2・汎用パーツは、特定のゲームのルールを知らない。

```
❌ 禁止：「ライフが0になったら敗北」を層2に実装
✅ 許可：「数値が0になった事実を記録」を層2に実装
```

### 2. 再利用可能性

汎用パーツは複数の利用場所を想定。

```
数値管理パーツ
  ├─ ゲームAでライフとして使用
  ├─ ゲームBでマナとして使用
  ├─ UIで表示用データとして使用
  └─ サーバーで同期用データとして使用
```

### 3. 責務分離

各要素は自分の責務のみを持つ。

```
層0が決めない：実装方法
層1が決めない：処理順序
層2が決めない：ゲームルール
汎用パーツが決めない：具体的な用途
```

### 4. イミュータビリティ

すべての操作が新しい状態を返す。

```
❌ オブジェクトを修正する
✅ 修正後の新しいオブジェクトを返す
```

## 設計思想：層外共有資産

汎用パーツは**層外共有資産**という新しい概念です。

```
従来：層0.5（曖昧な位置づけ）
    ↓
改善：層外共有資産（明確な位置づけ）
```

**意味：**
- 階層ではない
- すべての層が利用可能
- ゲーム非依存
- 複数利用場所を想定
- 再利用可能

この設計により、コードの再利用性が大幅に向上します。

---

## 重要な注意事項

### ゲーム固有ルールは層2以下に入らない

```
❌ 「ライフが0で敗北」を層2に実装
❌ 「毎ターン開始時にドロー」を層1に実装
❌ 「マナリジェネ」を汎用パーツに実装

✅ すべてゲーム側で実装する
```

### 汎用パーツは複数利用を前提

```
❌ 「ライフ管理」を汎用パーツに実装
✅ 「数値管理」を汎用パーツに実装
   （ゲーム側で「数値管理」をライフとして使用）
```

### イミュータビリティを保つ

```
❌ パーツの内部状態を修正
✅ 修正後の新しいパーツを返す
```
