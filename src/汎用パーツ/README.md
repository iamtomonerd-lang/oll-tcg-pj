# 汎用パーツ

複数のTCGで再利用可能な汎用的なゲーム要素の集合。

## 設計方針

### 汎用パーツが担当するもの

✅ 数値管理  
✅ プレイヤー情報管理  
✅ ゾーン機能  
✅ カード情報管理  
✅ 条件判定  
✅ その他汎用処理  

### 汎用パーツが含まないもの

❌ ゲーム固有ルール  
❌ 特定ゲームの名前  
❌ 勝敗条件判定  
❌ ターンシステム  
❌ 効果実装  

## 責任範囲の例

### 許可される設計

```typescript
// 数値管理パーツ
interface 数値 {
  現在値: number;
  最大値: number;
}

// ゲーム側で意味付け
// このパーツをライフとして使用
```

### 禁止される設計

```typescript
// ❌ ゲーム固有ルールを含む
interface ライフ {
  現在値: number;
  敗北条件: () => boolean;
}
```

## ゲーム側との関係

```
汎用パーツ（層0.5）
  └─ 汎用的な部品のみ提供
    ↓
ゲーム実装（層3+）
  └─ パーツにルール・名前を付ける
    └─ 具体的なゲーム化
```
