# 層0：ゲームに存在するもの

## 概要

層0は、TCGゲームの基本エンティティの型定義を行うレイヤーです。

「ゲームに何が存在するか」のみを定義し、「どのように動くか」や「何を意味するか」は上位層（層1以降）で定義されます。

## 定義するエンティティ

### ゾーン（Zone）
カード/トークンを置く場所を表します。

**関連ファイル:**
- `ゾーン/ゾーン.ts` - 型定義
- `ゾーン/ゾーン仕様.md` - 仕様書

**役割:**
- ゾーンの属性定義（名前、所有者、入るもの、公開領域、順番管理、移動可能性、数量制限）
- ゾーンは単なるコンテナで、エンティティの移動ルールは層2で定義

### カード（Card）
ゲームの基本エンティティです。

**関連ファイル:**
- `カード/カード.ts` - 型定義
- `カード/カード仕様.md` - 仕様書

**役割:**
- カードの基本構造（名前、種類、状態、数値情報、分類情報、効果）
- 「攻撃力」「コスト」などの具体的な意味は層2で定義
- 「横向き」「疲れた」などの状態名は層2で定義

### プレイヤー（Player）
ゲーム参加者を表します。

**関連ファイル:**
- `プレイヤー/プレイヤー.ts` - 型定義
- `プレイヤー/プレイヤー仕様.md` - 仕様書

**役割:**
- プレイヤーの基本構造（名前、状態、数値情報、分類情報、権限）
- 「ライフ」「マナ」などの具体的な意味は層2で定義
- プレイヤーはゾーンを所有せず、ゾーンが逆参照で所有者を管理

### ゲームオブジェクト（Game Object）
カード以外の存在（トークン、コアなど）を表します。

**関連ファイル:**
- `ゲームオブジェクト/ゲームオブジェクト.ts` - 型定義
- `ゲームオブジェクト/ゲームオブジェクト仕様.md` - 仕様書

**役割:**
- ゲームオブジェクトの基本構造（プレイヤーと同じ構造）
- 「トークン」「コア」などの具体的な種別は層2で定義

## 設計原則

### 不変性（Immutability）
層0の型は、上位層での操作を念頭に設計されています。実際の変更操作は層1で行われ、元のオブジェクトを変更せず、新しいオブジェクトを生成します。

### 汎用性（Generality）
層0はTCG依存性がありません。複数のTCGタイトルでも同じ型定義を使用できます。

### 責務分離（Separation of Concerns）
- **層0が定義すること:** エンティティが「何を持つことができるか」
- **層0が定義しないこと:** ゲームのルール、カード効果、ターン進行、勝敗判定、具体的な属性の意味

## 上位層との関係

### 層1（共通処理）
層0で定義されたエンティティに対する基本的な操作を提供します。
- ゾーン処理：エンティティの追加・取り出し・移動
- カード処理：カードの状態・数値・分類の変更
- プレイヤー処理：プレイヤーの状態・数値・分類の変更
- ゲームオブジェクト処理：ゲームオブジェクトの状態・数値・分類の変更

### 層2以降（TCG固有ルール）
具体的なゲームルールを実装します。
- 具体的な状態名（「配置済み」「アクティブ」など）
- 具体的な属性の意味（「ライフ」「パワー」など）
- カード効果の処理
- ターン進行・フェイズ管理
- 勝敗判定
