# 0層 カード構造仕様

## 定義の目的

カードという存在が持つことができる基本情報のみを定義する。

具体的なゲームルール、効果内容、数値の意味（「攻撃力」「コスト」など）は上位層で定義する。

## カードが持つ情報

```
カード
├ 名前
│   └ 型: 文字列
│
├ 種類
│   └ 型: 文字列
│
├ 状態
│   └ 型: 有り・無し
│
├ 数値情報
│   ├ 所持数
│   │   └ 型: 数字
│   └ 内容
│       └ 型: 所持数分の数字を配列で保持
│
├ 分類情報
│   ├ 所持数
│   │   └ 型: 数字
│   └ 内容
│       └ 型: 所持数分の文字を配列で保持
│
└ 効果
    └ 型: 有り・無し
```

## 各属性の説明

### 名前
- **型**: 文字列
- **説明**: カードの識別名
- **例**: 「火の剣」「治癒の光」「森のモンスター」
- **補足**: カード固有の名前は層3で定義される

### 種類
- **型**: 文字列
- **説明**: カードの分類
- **例**: 「魔法」「クリーチャー」「アーティファクト」
- **補足**: 具体的な種類は上位層で定義される

### 状態
- **型**: 有り / 無し
- **説明**: カードが状態を持つか
- **例**:
  - 有り：カードが「横向き」「疲れた」などの状態を持つ可能性がある
  - 無し：カードが状態を持たない
- **補足**: 具体的な状態名（「横向き」「疲れた」など）は層2以上で定義される

### 数値情報
- **構造**:
  - 所持数：数字（何種類の数値を保持するか）
  - 内容：所持数分の数字を配列で保持
- **説明**: カードが複数の数値を持つことができる
- **例**:
  - 所持数: 2
  - 内容: [5, 3]
  - （意味は層2以上で決定。例：[攻撃力, コスト]）
- **補足**: 数値の具体的な意味は層2以上で定義される

### 分類情報
- **構造**:
  - 所持数：数字（何種類の分類を保持するか）
  - 内容：所持数分の文字を配列で保持
- **説明**: カードが複数の分類タグを持つことができる
- **例**:
  - 所持数: 3
  - 内容: ["火", "攻撃型", "レア"]
  - （意味は層2以上で決定。例：[属性, タイプ, レアリティ]）
- **補足**: 分類の具体的な意味は層2以上で定義される

### 効果
- **型**: 有り / 無し
- **説明**: カードが効果を持つか
- **例**:
  - 有り：カードが何らかの効果を持つ可能性がある
  - 無し：カードが効果を持たない
- **補足**: 効果の具体的な内容は層2以上で定義される

## 記述例

### サンプルTCG：「火の剣」
```
名前: 火の剣
種類: 武器
状態: 有り
数値情報:
  所持数: 2
  内容: [5, 3]
分類情報:
  所持数: 2
  内容: ["火", "レア"]
効果: 有り
```
（意味は層2で定義。例：数値は[攻撃力, コスト]、分類は[属性, レアリティ]）

### サンプルTCG：「基本スライム」
```
名前: 基本スライム
種類: クリーチャー
状態: 無し
数値情報:
  所持数: 1
  内容: [2]
分類情報:
  所持数: 1
  内容: ["ノーマル"]
効果: 無し
```

### サンプルTCG：「ターンスキップ」
```
名前: ターンスキップ
種類: 魔法
状態: 無し
数値情報:
  所持数: 0
  内容: []
分類情報:
  所持数: 1
  内容: ["瞬間"]
効果: 有り
```

## 可変性の特徴

### 数値情報の可変性
- 異なるカードで異なる数の数値を持つことが可能
- 例：
  - カードA：2個の数値 [5, 3]
  - カードB：3個の数値 [10, 2, 8]
  - カードC：0個の数値 []

### 分類情報の可変性
- 異なるカードで異なる数の分類を持つことが可能
- 例：
  - カードA：2個の分類 ["火", "レア"]
  - カードB：3個の分類 ["水", "ノーマル", "攻撃型"]
  - カードC：1個の分類 ["風"]

## 層0が定義しないもの

- **具体的な数値の意味**
  - 例：「最初の数値は攻撃力」は層2で定義
  
- **具体的な分類の意味**
  - 例：「最初の分類は属性」は層2で定義

- **具体的な状態の種類**
  - 例：「横向き状態」は層2で定義

- **具体的な効果の内容**
  - 例：「このカードを使用するとダメージを与える」は層2で定義

- **カード個別の設定値**
  - 例：個別カードの攻撃力や効果テキストは層3で定義

## 設計の利点

1. **柔軟性**: 異なるゲームシステムに対応可能
2. **拡張性**: 新しいゲーム要素の追加が容易
3. **保守性**: 層間の責任が明確に分離されている
4. **理解性**: カード構造の本質が明確
