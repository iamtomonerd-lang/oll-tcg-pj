# 0層 ゾーン仕様

## 定義の目的

ゾーンが持つことができる情報の種類のみを定義する。

具体的なルール（「手札は7枚まで」など）、移動先、効果、数値などは上位層で定義する。

## ゾーンが持つ情報

```
ゾーン
├ 名前
│   └ 型: 文字列
│
├ 所有者
│   └ 値: 自分・相手・共有
│
├ 入るもの
│   └ 値: カード・トークン
│
├ 公開領域
│   └ 値: 全員・いいえ・所有者・指定者
│
├ 順番管理
│   └ 値: はい・いいえ
│
├ 移動可能
│   └ 値: はい・いいえ
│
└ 数量制限
    └ 値: 数字・いいえ
```

## 各属性の説明

### 名前
- **型**: 文字列
- **説明**: ゾーンの識別名
- **例**: 「手札」「場」「山札」「トラッシュ」
- **補足**: 具体的な名前は上位層で決定される

### 所有者
- **値**: 自分 / 相手 / 共有
- **説明**: ゾーンの所有者
- **例**:
  - 自分：プレイヤー1の手札
  - 相手：プレイヤー2の手札
  - 共有：ゲーム全体の共有ゾーン

### 入るもの
- **値**: カード / トークン
- **説明**: ゾーンに置くことができるエンティティの種類
- **補足**: ゲームではカードとトークンの2種類が基本

### 公開領域
- **値**: 全員 / いいえ / 所有者 / 指定者
- **説明**: ゾーンの情報を誰が見えるか
- **例**:
  - 全員：場のカードは全員に見える
  - いいえ：手札は見えない（所有者のみ）
  - 所有者：所有者のみが見える
  - 指定者：特定の対象者のみが見える

### 順番管理
- **値**: はい / いいえ
- **説明**: ゾーン内のカード順序が意味を持つか
- **例**:
  - はい：山札は順序が重要
  - いいえ：手札は順序が意味を持たない場合もある

### 移動可能
- **値**: はい / いいえ
- **説明**: ゾーン自体が移動可能か
- **例**:
  - はい：移動可能なゾーン
  - いいえ：固定されたゾーン

### 数量制限
- **型**: 数字 / いいえ
- **説明**: ゾーンに入るエンティティの数に制限があるか
- **例**:
  - 7（数字）：最大7枚まで入れられる
  - いいえ：制限なし

## 注記

- この仕様は「構造」のみを定義し、「制限ルール」は定義しない
- 例：「手札は7枚まで」という具体的な制限は層2で定義される
- ゾーン自体が何を持つことができるかのみを定義
