# 0層 ゲームオブジェクト構造仕様

## 定義の目的

ゲームオブジェクトという存在が持つことができる基本情報のみを定義する。

ゲームオブジェクトとは、カードではないがゲーム内に存在するもの全般を指す。

具体的なゲームオブジェクトの種類や意味は上位層で定義する。

## ゲームオブジェクトの範囲

**ゲームオブジェクトに含まれるもの:**
- トークン
- コア
- カウンター
- コイン
- 紋章
- その他カードではないゲーム上の存在

**ゲームオブジェクトに含まれないもの:**
- カード（CARD_SPEC.mdで定義）
- プレイヤー（PLAYER_SPEC.mdで定義）
- ゾーン（ZONE_SPEC.mdで定義）

## ゲームオブジェクトが持つ情報

```
ゲームオブジェクト
├ 名前
│   └ 型: 文字列
│
├ 状態
│   ├ 所持数
│   │   └ 型: 数字
│   └ 内容
│       └ 型: 所持数分の文字を配列で保持
│
├ 数値情報
│   ├ 所持数
│   │   └ 型: 数字
│   └ 内容
│       └ 型: 所持数分の数字を配列で保持
│
├ 分類情報
│   ├ 所持数
│   │   └ 型: 数字
│   └ 内容
│       └ 型: 所持数分の文字を配列で保持
│
└ 効果
    └ 型: 有り・無し
```

## 各属性の説明

### 名前
- **型**: 文字列
- **説明**: ゲームオブジェクトの識別名
- **例**: 「トークン」「コア」「カウンター」「コイン」
- **補足**: 具体的な名前は層3で定義される

### 状態
- **構造**:
  - 所持数：数字（何種類の状態を持つか）
  - 内容：所持数分の文字を配列で保持
- **説明**: ゲームオブジェクトが複数の状態を持つことができる
- **例**:
  - 所持数: 1
  - 内容: ["配置済み"]
  - （意味は層2以上で決定。例：[配置状態]）
- **補足**: 具体的な状態名は層2以上で定義される

### 数値情報
- **構造**:
  - 所持数：数字（何種類の数値を保持するか）
  - 内容：所持数分の数字を配列で保持
- **説明**: ゲームオブジェクトが複数の数値を持つことができる
- **例**:
  - 所持数: 2
  - 内容: [5, 3]
  - （意味は層2以上で決定。例：[パワー, レベル]）
- **補足**: 数値の具体的な意味は層2以上で定義される

### 分類情報
- **構造**:
  - 所持数：数字（何種類の分類を保持するか）
  - 内容：所持数分の文字を配列で保持
- **説明**: ゲームオブジェクトが複数の分類タグを持つことができる
- **例**:
  - 所持数: 2
  - 内容: ["火", "トークン"]
  - （意味は層2以上で決定。例：[属性, タイプ]）
- **補足**: 分類の具体的な意味は層2以上で定義される

### 効果
- **型**: 有り・無し
- **説明**: ゲームオブジェクトが効果を持つか
- **例**:
  - 有り：ゲームオブジェクトが何らかの効果を持つ可能性がある
  - 無し：ゲームオブジェクトが効果を持たない
- **補足**: 効果の具体的な内容は層2以上で定義される

## ゲームオブジェクトの設計特性

### カードとの違い

| 属性 | カード | ゲームオブジェクト |
|------|--------|-------------------|
| 名前 | あり | あり |
| 種類 | あり | なし（分類で表現） |
| 状態 | 有り・無し | 複数の状態を保持可 |
| 数値情報 | あり（可変長） | あり（可変長） |
| 分類情報 | あり（可変長） | あり（可変長） |
| 効果 | 有り・無し | 有り・無し |

### 可変性の特徴

#### 状態の可変性
- 異なるゲームオブジェクトで異なる数の状態を持つことが可能
- 例：
  - オブジェクトA：1個の状態 ["配置済み"]
  - オブジェクトB：2個の状態 ["配置済み", "アクティブ"]
  - オブジェクトC：0個の状態 []

#### 数値情報の可変性
- 異なるゲームオブジェクトで異なる数の数値を持つことが可能
- 例：
  - オブジェクトA：1個の数値 [1]
  - オブジェクトB：2個の数値 [5, 3]
  - オブジェクトC：0個の数値 []

#### 分類情報の可変性
- 異なるゲームオブジェクトで異なる数の分類を持つことが可能
- 例：
  - オブジェクトA：1個の分類 ["トークン"]
  - オブジェクトB：2個の分類 ["火", "コア"]
  - オブジェクトC：0個の分類 []

## 層0が定義しないもの

- **具体的なゲームオブジェクトの種類**
  - 例：「トークン」「コア」は層2で定義

- **具体的な状態の種類**
  - 例：「配置済み」「アクティブ」は層2で定義

- **具体的な数値の意味**
  - 例：「パワー」「レベル」は層2で定義

- **具体的な分類の意味**
  - 例：「属性」「タイプ」は層2で定義

- **具体的な効果の内容**
  - 例：「エネルギー供給」は層2で定義

## 設計の利点

1. **汎用性**: カード・プレイヤー・ゾーンではない全てのゲーム内存在に対応
2. **柔軟性**: 異なるゲームシステムに対応可能
3. **拡張性**: 新しいゲーム要素の追加が容易
4. **保守性**: 層間の責任が明確に分離されている
5. **一貫性**: カード・プレイヤーと同じ構造パターンを使用

## 重要な注記

ゲームオブジェクトは「カードではないゲーム内の存在」を表す共通概念である。

今後どのTCGを実装しても、カードではない存在はゲームオブジェクトとして扱う。

カード固有の情報やルール、ゲームタイトル固有の概念はここへ追加してはいけない。

全て下位層で定義すること。
