# OLL TCG エンジン

複数のトレーディングカードゲーム（TCG）を同じ基盤で実装できる汎用ゲームエンジンです。

## プロジェクト理念

このプロジェクトの目的は3つあります。

### 1. 複数のTCGタイトルを実装できる汎用エンジン

特定のカードゲーム専用のシステムではなく、
将来的に様々なTCGを同じ基盤で実装できる構造を目指します。

### 2. 高度なAI対戦を可能にすること

このエンジンはAIがカードゲームを理解し、
強い対戦相手になることを目的とします。

ゲーム状態、プレイヤーの行動、カード情報などが
AIにとって解析可能な形で管理されます。

### 3. カードゲーム経験者が読めるコード

プログラミング知識よりもカードゲーム知識を重視します。

可能な限り日本語のカードゲーム用語を使用し、
コードを見るだけでカードゲームの仕様が理解できることを目標にします。

## 層構造

このプロジェクトの処理は責任範囲ごとの「層」に分割されます。

```
層0：ゲームに存在するもの
  ↓
層1：共通処理
  ↓
層2：TCG個別ルール
  ↓
層3：カード定義
```

### 重要：層番号は上下関係ではなく、責任の大きさを表します

- **数字が小さい層** → 責任が大きい、抽象的
- **数字が大きい層** → 責任が小さい、具体的

### 層の依存ルール

- 小さい番号の層は大きい番号の層へ影響を与える（許容）
- 大きい番号の層は小さい番号の層を変更してはいけない（禁止）

## 層の役割

### 層0：ゲームに存在するもの

ゲームに存在する基本的なエンティティの「構造」を定義します。

**定義するもの:**
- ゾーン（カード/トークンを置く場所）
- カード（ゲームの基本エンティティ）
- プレイヤー（ゲーム参加者）
- ゲームオブジェクト（カード以外の存在）

**定義しないもの:**
- ゲームルール
- 処理ロジック
- 特定TCG固有の概念（ライフ、マナ、コストなど）

### 層1：共通処理（未実装）

TCG全体に共通する処理を実装します。

例：ゾーン間のカード移動、ターン進行など

### 層2：TCG個別ルール（未実装）

特定のTCGタイトルに固有のゲームルールを定義します。

### 層3：カード定義（未実装）

個別のカード情報を定義します。

## 開発方針

- **日本語優先**：カードゲーム用語をそのまま使用
- **型安全性**：TypeScriptの厳密な型定義
- **責務の明確化**：各層の責任範囲を明確に保つ
- **拡張性**：複数TCG対応を想定した設計
